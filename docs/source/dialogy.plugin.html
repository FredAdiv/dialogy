dialogy.plugin.plugin module¶

Module provides access to an abstract plugin class.

Summary

We will summarize a few key points for creating plugins:

  • Don’t interact with the workflow directly, use functions to access and mutate.

  • The convention for workflow access is access(workflow).

  • The convention for workflow modification is mutate(workflow, value).

  • Plugin names must end with Plugin for classes and _plugin for functions. examples: Sentence2VecPlugin, words2num_plugin.

class dialogy.plugin.plugin.Plugin(access: Optional[Callable[[…], Any]], mutate: Optional[Callable[[…], Any]])[source]¶

Bases: object

A plugin object interacts with a workflow. Workflows expect a set of plugins to be inserted into different stages: pre and post processing. A plugin can be conveniently written being unaware of the structure of a given [workflow](../workflow/workflow.html) by expecting access and mutate functions.
